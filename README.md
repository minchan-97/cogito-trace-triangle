# cogito_trace — AI를 믿기 위한 근거 소급 + 분기 추적

## 두 축
- **세로축 (cogito_trace.py)**: 답변의 근거를 앵커(인간 보증/공리 문서)까지 소급.
  각 연결의 실재를 검증해 '지어낸 근거'를 차단. → grounded/weak/ungrounded.
- **가로축 (divergence_trace.py)**: 여러 LLM/생성의 근거 사슬을 정렬해,
  '어느 마디에서 갈라지는가'(분기점)를 탐지.
  - 말단 분기 → 표현 차이(사소)
  - 앵커 근처 분기 → 근본 근거가 다름(심각)

## 핵심 발상
최종 불일치율(한 숫자)이 아니라, **사슬 위 분기 위치**로 신뢰를 판단한다.
"어디서 갈라지나"가 "왜 못 믿나"를 정확히 짚는다.

## 벡터 기록
일치/불일치/환각 문장의 벡터를 전량 기록(pkl).
라벨별 응집도 + 라벨 간 거리로 패턴을 짚기 위함.
→ 실제 OpenAI 임베딩(embed_fn 주입) 시 진짜 분포가 드러남.

## 정직한 범위
- 'LLM 내부 사고'가 아니라 'LLM이 제시한 근거 사슬'의 추적.
  단 각 연결 실재를 검증하므로 가짜 근거는 사슬에서 끊김.
- 임베딩 미주입 시 해시 폴백(구조 검증용, 의미 없음).
- 사슬 정렬/표현-근거 구분은 실제 임베딩 + 임계값 튜닝 필요.

## claim 대조 통합 (v2)
오늘 3축(유사도/근거/극성)에 어제 claim 대조(주어-서술 일치)를 통합.
4가지 오류 이유로 판정:
  - MATCH: 주어·서술 모두 일치
  - CONTRADICTION(모순): 자료를 부정으로 뒤집음 (~이다↔아니다)
  - VALUE_ERROR(값오류): 서술은 유사하나 주어 교체됨(치즈버거≠세징야), 또는 항목은 있으나 값 다름
  - HALLUCINATION(환각): 근거 자체가 자료에 없음

각 판정이 곧 '왜 틀렸는가'의 설명. 실제 임베딩(OpenAI) 주입 시 정확.
⚠️ 극성/부정/주어 판정은 규칙 기반 — 명백한 케이스 위주. 임계값 튜닝 필요.
